Step 1: Imagine three simple functions
function double(x) {
return x \* 2;
}

function addThree(x) {
return x + 3;
}

function square(x) {
return x \* x;
}

If you call them normally:

square(addThree(double(2)))

The computer does this:

2

↓

double

↓

4

↓

addThree

↓

7

↓

square

↓

49
Step 2: The problem

Writing this gets messy:

square(addThree(double(2)))

What if you had 10 functions?

f10(f9(f8(f7(f6(f5(...)))))

😵 That's hard to read.

Step 3: What do we want?

Instead of writing

square(addThree(double(2)))

we want

const calculate = compose(
square,
addThree,
double
);

calculate(2);

Notice something important:

compose(...) doesn't calculate anything immediately.

It creates a new function.

Step 4: Think about what compose receives
compose(
square,
addThree,
double
)

It receives...

square

addThree

double

Three functions.

Not numbers.

Step 5: What should it return?

It should return another function.

Think:

Functions

↓

compose

↓

NEW function

So your skeleton looks like this:

function compose(/_ functions _/) {

    return function(value) {

    }

}
